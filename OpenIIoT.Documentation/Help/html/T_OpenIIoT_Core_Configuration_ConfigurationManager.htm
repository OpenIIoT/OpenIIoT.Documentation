Remarks

The Configuration file is generated from the json serialization of the Configuration model. The Configuration model consists of a single instance of Configuration. This instance creates a nested dictionary keyed on type first, then by name. The resulting key value pair contains the Configuration object for the specified Type and named instance.

There are two main types of configuration supported by this scheme; configuration for static objects like the various application Managers and Services, and for dynamic objects encompassed by Plugins. The key difference is the number of instances of each; static objects will have only one instance while Plugin objects may have any number. Static objects do not supply an instance name when using the Configuration Manager, and their configuration is saved within the model with an empty string. Dynamic objects must supply an instance name.

The Configuration file maintained by the Configuration Manager is capable of being rebuilt from scratch. If missing, the Manager automatically adds a default, nameless instance of each registered type to the configuration model and flushes it to disk before loading it. This ensures the application can start normally in the event of a deletion.

A Type is considered to be configurable if:

  1. it implements the IConfigurableT interface
  2. It contains the static method GetConfigurationDefinition()
If both predicates are true, the Type can be registered with the Configuration Manager and instances of that type can load and save configuration data.

Before any Type can use the Configuration Manager, the method RegisterType(Type, Boolean) method within the ConfigurableTypeRegistry must be called and passed the Type of that class. This method fetches the ConfigurationDefinition for the Type from the static method GetConfigurationDefinition and stores the Type and the ConfigurationDefinition in the RegisteredTypes dictionary.

Configured objects may retrieve their stored configuration via the GetInstanceT(Type, String) method, and may save their configuration using UpdateInstance(Type, Object, String).

New instances of configurable Types may be added to the configuration using the AddInstanceT(Type, Object, String) method, and may be removed using RemoveInstance(Type, String). The IsInstanceConfigured(Type, String) method is used to determine whether a given instance is configured.

The configuration is loaded from file within the Startup method, and saved within Shutdown(StopType). The filename of the stored configuration is defined in the application settings file with the key "ConfigurationFileName". If the key is missing from the configuration, the default name "OpenIIoT.json" is used.
