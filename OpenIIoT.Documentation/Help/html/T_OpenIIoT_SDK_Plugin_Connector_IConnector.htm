Remarks

Classes implementing IConnector are responsible for application extensibility, providing read/write access to external data stores and interfaces.

All data exposed by an IConnector must be represented as an Item with instance type [!:ConnectorItem]. ConnectorItem instances must be arranged in a composite (tree) data structure, the minimum structure being that of a single root node. This data structure is accessed by the application with the [!:Browse()] and [!:Browse(Item)] methods. The [!:Find(string)] method must be implemented to search for and return an Item matching the specified Fully Qualified Name within the data structure, if it exists.

Asynchronous overloads are provided for each of the methods. These are: [!:BrowseAsync()], [!:BrowseAsync(Item)], and [!:FindAsync(string)]. Other than enclosing the return type in a TaskTResult and executing asynchronously, these methods are identical to their synchronous counterparts.

The IConnector interface represents the minimun implementation of the class. Other functionality may be implemented by implementing one or more of the following interfaces: ISubscribable: Allows the application to subscribe Model Items to ConnectorItems to receive value updates. IWriteable: Allows the application to write values to the source of a ConnectorItem.IExtensible: Allows the application to add additional ConnectorItems to the Connector at runtime.
