Remarks

An [!:IConnector] instance implementing IExtensible is responsible for adding additional [!:ConnectorItem] s to the Connector Plugin's Item data structure at runtime. This functionality is intended primarily for data sources without the capability of fully describing the Items that are available for reading.

New ConnectorItems are added via the AddItem(String, String) method. This method accepts a string containing the Fully Qualified Name for the new item, and a string containing the Fully Qualified Source Name for the item. The Source Name should be validated by the Plugin to ensure it is well formed. The Add() method must return an Result containing the result of the operation and Item that was added, as well as any informational, warning or error messages that were generated during the operation.

Fully Qualified Names may be any well formed name. If the path of the FQN contains items that don't yet exist, the Plugin is responsible for creating them. For example, if the specified FQN is "Devices.Switches.Momentary.Paddle" and only the "Devices" Item exists, the Add() method is required to create "Switches", "Momentary" and "Paddle", with the SourceFQN property of "Paddle" set to the specified Fully Qualified Source Name.

A list (rather, dictionary) of Item s added using the AddItem(String, String) method shall be made available in the AddedItems property. This property is a DictionaryTKey, TValue, keyed on the Fully Qualified Name of the Item and with a value of the Source Fully Qualified Name.

The AddedItems list must be persisted and each added Item must be created and added to the appropriate location in the Connector's Item data structure when the Connector is initialized. Should the automatically populated Item structure within the Connector change such that a user defined Item's parent(s) no longer exist, the Item must still be added to the structure in accordance with the AddItem(String, String) method (that is, the structure must be created if it does not exist).
